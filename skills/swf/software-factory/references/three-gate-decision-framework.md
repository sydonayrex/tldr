# Three-Gate Decision Framework (Language Standard)

## Decision Language Convention

The Value Review Gate uses specific language for its three decision points. This is intentional and MUST be applied consistently across all factory skills, metrics, and outputs.

| Gate | STOP Decision | CONTINUE Decision | Rationale |
|------|--------------|-------------------|-----------|
| Gate 1 (Intake) | **PUNT** | BUILD | "Punt" implies a choice to stop, not destruction |
| Gate 2 (Development) | **PUNT** | **PERSEVERE** | "Persevere" implies conviction, not inertia |
| Gate 3 (Ship) | DECOMMISSION | **PERSEVERE** | "Persevere" for continuing in production |

## Rules

1. **PUNT, not KILL**: When a bet or product is stopped, say "punt". "Kill" implies the work was wasted; "punt" implies a strategic choice to redirect.
2. **PERSEVERE, not CONTINUE**: When a bet or product continues, say "persevere". "Continue" implies inertia; "persevere" implies active conviction.
3. **DEFER, not DELAY**: When pushing to backlog intentionally. "Delay" implies failure; "defer" implies conscious prioritization.
4. **DECOMMISSION, not DELETE**: When removing from production. "Delete" is a technical action; "decommission" is a product decision (includes user notification, data migration, sunset plan).

## Apply In

- Value Review Gate output templates
- Gate 1/2/3 decision tables
- Metrics ("Gate 1 punt rate", not "Gate 1 kill rate")
- Decision logs ("PUNTED: [feature] because [reason]")
- Orchestration protocol language
- CPO skill value reports
- Cross-cutting concerns table
