# Design System Strategy Integration Notes

## Source
Public research on design system consulting patterns. Synthesized into Product Design v2.0. Source NOT referenced in any skill per user directive.

## Key Patterns Integrated

### The Squish Map Approach
Every design system term has a "general definition" AND a "squish zone" (what it might also mean in different contexts). The practitioner's job is not to be"right" about definitions but to recognize squishiness and firm it up for your team.

### Terminology Is the First Action Item
If the team doesn't share a common language for design system terms, every subsequent conversation will be misaligned. Norm on terminology BEFORE doing any other design system work.

### Three-Tier Token Architecture (DTCG)
1. Primitive (raw values)
2. Semantic (meaning)
3. Component (specific mapping)

### Four-Layer Value Cake
Design systems deliver value at: Organizational, Team, Individual Practitioner, and User levels.

### Adoption Decision Matrix
Four approaches: Adopt, Adapt, Build, Bypass. Each has clear trade-offs documented in the skill.

### Governance Is Not Optional
Contribution process, deprecation path, versioning, ownership, adoption tracking, feedback loop, quality bar. All required for a system to survive beyond initial creation.
