# PM Playbook Integration Notes

## Source
Product management best-practices playbook (multi-hundred-page PDF). Synthesized into Product Management v3.0 and CEO v2.0. Source NOT referenced in any skill per user directive.

## Key Patterns Integrated

### Vision → Strategy → Roadmap Cascade
- Vision = WHY (aspirational, 5+ years, user-focused, solution-agnostic, unique)
- Strategy = HOW (choices about where to invest AND what to say no to)
- Roadmap = WHAT NEXT (outcomes across three horizons: Current, Near Term, Future)

### Outcome-Oriented Planning
- Goals (long-term, strategic) → Objectives (SMART) → KPIs (actionable ratios)
- Vanity metric test: "If this number changes, will we do something differently?"

### Hypothesis-Driven Experimentation
- We believe <capability> will result in <outcome>
- Test it, measure it, set validation criteria before running
- One variable per hypothesis

### MVP = Learning Milestone
- Not a delivery milestone. Not a proof of concept.
- Smallest thing that enables maximum validated learning with least effort.

### Vertical Slices
- One complete user-facing feature at a time, not horizontal layers
- Smaller slices = better estimation, faster feedback, lower risk

### Story Format
- Connextra: As a [user], I want [goal], so that [benefit]
- Gherkin acceptance criteria: Given/When/Then

### Value-Quality-Constraints
- You can fix two of three. Agile teams fix time; value and quality flex.
