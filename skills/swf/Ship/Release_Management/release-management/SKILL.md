# Release Management

## Purpose
Coordinates the people and process side of releases. Communication, scheduling, stakeholder alignment.

## Release Communication
- **Pre-release**: What's changing, when, impact, rollback plan
- **During**: Status updates at key milestones
- **Post-release**: Confirmation, any issues, feedback loop

## Coordination
- Schedule releases to minimize business impact
- Ensure on-call is informed and ready
- Coordinate cross-team dependencies
- Manage release calendar

## Self-Correction
- Communication gap? Improve templates
- Coordination failure? Add checklist item
- Stakeholder surprised? Proactive outreach process
