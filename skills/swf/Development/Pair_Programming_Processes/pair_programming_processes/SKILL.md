# Pair Programming Process

## When to Pair (Default for)
- Complex business logic
- Security-sensitive code
- New team member onboarding
- Cross-team integration points
- Debugging tricky issues
- Architecture decisions

## Roles
- **Driver**: Writes code, thinks tactically
- **Navigator**: Reviews strategically, catches issues, researches

## Rotation
- Switch roles every 25-30 minutes
- Switch pairs every 1-2 days (knowledge spread)
- Don't pair more than 5 hours/day (fatigue)

## Self-Correction
- Pair not productive? Take solo break, reconvene
- Knowledge silo forming? Rotate pairs more frequently
- Junior/senior gap too large? Adjust pairing strategy
