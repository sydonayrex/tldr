# QA Process

## QA in Each Phase
- **Shaping**: Review testability, define acceptance criteria
- **Development**: Write automation, coach developers on testing
- **Pre-release**: Execute exploratory testing, verify critical paths
- **Post-release**: Monitor production quality metrics

## Bug Triage
- Daily triage of new bugs
- Severity assignment (S1-S4)
- Assignment to team/sprint
- Verification of fixes

## Self-Correction
- Bug escape? Add test, improve process
- Test environment unreliable? Fix infrastructure
- Coverage gap? Prioritize high-risk areas
