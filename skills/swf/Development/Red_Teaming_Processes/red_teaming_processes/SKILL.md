# Red Teaming Process

## What We Red Team
- New features (before launch)
- Architecture decisions (before commitment)
- Assumptions in shaped bets (during intake)
- Production systems (periodic exercises)

## Red Team Questions
- How would an attacker abuse this?
- What happens if this assumption is wrong?
- What's the worst-case failure mode?
- What are we NOT testing?
- What would a malicious insider do?

## Self-Correction
- Red team finding? Fix before launch
- Assumption proven wrong? Pivot, don't defend
- Red team skipped? Retroactive review
